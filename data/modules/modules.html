модули

модуль - это просто файл

        export function foo() {...}
        import {foo} from './path/file.js'

модули нужно указывать

script type="module" src='file.js'> /script>

модули не работают локально, через протокол file://. Только через сервер

модули всегда рабоют через 'use strict'

модули имеют свою область видимости

при изменении в экспортируемом модуле свойств, то во всех импортируемых модулях видны эти изменения

export

экспорт отдельных фрагментов 

        export let arr = [1, 2, 3];
        export function foo() {...}
        export class User{...}

экспорт отдельно от объявления

        function foo() {...}
        function foo2() {...}

        export {foo, foo2};

импорт

import {foo, foo2} from './file.js'

импорт всего файла

import * as module from './module.js'

экспорт по умолчанию

разница

именованный экспорт - export {}

        export class User {...}
        import {User} from './.../file.js'

экспорт по умолчанию - export default

        export default class User {...}
        import User from './.../file.js'

отличия

  1. Количество экспортируемых значений: export default используется для экспорта только одного значения из модуля, тогда как export {} позволяет экспортировать несколько именованных значений.
  2. Импортирование: При импорте значений, экспортированных с помощью export default, вы можете дать этому значению любое имя. При импорте именованных экспортов через export {}, вы должны использовать точное имя экспортируемого значения.
  3. Применение: export default чаще используется для экспортирования главной функции или класса модуля, тогда как export {} подходит для экспортирования нескольких утилитарных функций или констант.
